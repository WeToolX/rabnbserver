# 管理员充值接口对接文档

## 1. 接口说明

管理员给指定用户内部余额充值，并生成用户账单记录。

- 接口地址：`POST /api/admin/user/amount/deposit`
- 请求方式：`POST`
- 请求类型：`application/json`
- 鉴权方式：Sa-Token 登录校验
- Token 请求头：`Account-token: {token}`
- 控制器位置：`src/main/java/com/ra/rabnbserver/controller/user/AdminUserController.java`

## 2. 请求参数

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `userId` | `Long` | 是 | 被充值用户 ID |
| `amount` | `String` | 是 | 充值金额，必须是大于 0 的数字字符串 |
| `transactionType` | `String` | 否 | 交易业务类型，传枚举 code；不传时默认 `DEPOSIT` |
| `source` | `String` | 否 | 备注。传入后作为账单备注；不传时默认 `系统充值` |

> 注意：`source` 只作为备注使用，不再用于判断交易类型。交易类型由 `transactionType` 字段决定。

## 3. 交易类型枚举

`transactionType` 支持传下面任一枚举 code。建议前端固定传大写 code。

| code | 中文说明 | 适用说明 |
| --- | --- | --- |
| `PURCHASE` | 购买 | 购买类账单 |
| `SELL` | 卖出 | 卖出类账单 |
| `DEPOSIT` | 充值 | 普通充值，接口默认值 |
| `WITHDRAWAL` | 提现 | 提现类账单 |
| `EXCHANGE` | 闪兑 | 闪兑类账单 |
| `REWARD` | 奖励 | 人工奖励、绩效奖励等 |
| `PROFIT` | 收益 | 收益入账 |
| `TRANSFER` | 转账 | 转账类账单 |
| `MINER_ELECTRICITY` | 矿机电费 | 矿机电费相关账单 |
| `GOLD_QUANT` | 黄金量化 | 黄金量化业务账单 |
| `GOLD_QUANT_REWARD` | 黄金量化奖励分成 | 黄金量化奖励分成 |
| `GOLD_QUANT_DISTRIBUTION` | 黄金量化分销分成 | 黄金量化分销分成 |

枚举反序列化支持大小写不敏感，例如 `reward`、`REWARD` 都可以识别为 `REWARD`。为了避免歧义，前端仍建议统一传大写 code。

## 4. 请求示例

### 4.1 普通系统充值

```json
{
  "userId": 10001,
  "amount": "100",
  "transactionType": "DEPOSIT",
  "source": "系统充值"
}
```

### 4.2 人工奖励/绩效充值

```json
{
  "userId": 10001,
  "amount": "88.5",
  "transactionType": "REWARD",
  "source": "v1 团队电力绩效"
}
```

### 4.3 curl 示例

```bash
curl -X POST 'https://{host}/api/admin/user/amount/deposit' \
  -H 'Content-Type: application/json' \
  -H 'Account-token: {admin_token}' \
  -d '{
    "userId": 10001,
    "amount": "88.5",
    "transactionType": "REWARD",
    "source": "v1 团队电力绩效"
  }'
```

## 5. 返回结构

统一返回 `ApiResponse` JSON。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | `Integer` | 业务状态码，`200` 表示成功 |
| `message` | `String` | 提示信息 |
| `data` | `Any` | 业务数据 |

### 5.1 成功返回

```json
{
  "code": 200,
  "message": "操作成功",
  "data": "充值成功"
}
```

### 5.2 失败返回示例

充值金额为空：

```json
{
  "code": 401,
  "message": "充值金额不能为空",
  "data": null
}
```

充值金额格式错误：

```json
{
  "code": 401,
  "message": "充值金额格式错误",
  "data": null
}
```

充值金额小于等于 0：

```json
{
  "code": 401,
  "message": "充值金额必须大于0",
  "data": null
}
```

业务处理失败：

```json
{
  "code": 401,
  "message": "充值失败: {失败原因}",
  "data": null
}
```

## 6. 业务规则

- `amount` 会使用 `BigDecimal` 解析，必须能被解析为数字。
- `amount <= 0` 时拒绝充值。
- `transactionType` 不传时默认使用 `DEPOSIT`。
- `source` 有值时作为账单备注；没有值时备注为 `系统充值`。
- 该接口固定生成平台账单：`BillType.PLATFORM`。
- 该接口固定为入账类型：`FundType.INCOME`。
- 平台内充值没有链上交易哈希，`txHash` 为空。
